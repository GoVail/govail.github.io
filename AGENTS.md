# GoVail Site — AI Collaboration Guide

이 가이드는 GoVail 공식 사이트(`govail.github.io`) 프로젝트에 기여하는 개발자 및 AI 에이전트가 준수해야 할 작업 표준, 콘텐츠 원칙, 그리고 보안 규칙을 정의합니다.

---

## 1. 프로젝트 개요

**GoVail Site**는 GoVail 플랫폼의 공식 웹사이트로, VitePress 기반 정적 사이트 생성기를 사용합니다.

```text
govail.github.io/
├── index.md           — 메인 랜딩 페이지
├── posts/             — 블로그 포스트 (기술 문서, 릴리즈 노트)
└── .vitepress/        — VitePress 설정 및 테마
```

---

## 2. 콘텐츠 작성 원칙

- **오픈소스 톤**: 과장된 마케팅 표현을 지양하고, 기술적 사실에 근거한 차분하고 실용적인 톤을 유지한다.
- **정보 분리**: 사내 인프라 정보, 실제 IP, 내부 서비스명은 절대 공개 콘텐츠에 포함하지 않는다.
- **Mermaid 지원**: 기술 다이어그램은 Mermaid를 사용하며, VitePress에서 렌더링됨을 확인한다.
- **SEO 준수**: 모든 페이지에 `title`, `description` 메타 태그를 포함한다.

---

## 3. 보안 및 민감 정보 노출 방지 규칙 (DLP)

공개 웹사이트이므로 아래의 정보 포함을 엄격히 금지합니다:

- **호스트 상세**: 내부 서버의 실제 호스트네임, 사설 IP 주소 대역, VPN 정보.
- **운영 데이터**: 실제 고객사 명칭, 개인식별정보(PII).
- **자격 증명**: API 키, 토큰 등 어떠한 형태의 인증 정보.
- **빌드 산출물**: `node_modules/`, `.vitepress/dist/` 등.

---

## 4. 개발 워크플로우

```bash
# 의존성 설치
npm install

# 로컬 개발 서버 (라이브 미리보기)
npm run dev

# 빌드 검증 (배포 전)
npm run build
```

---

## 5. 버전 관리 및 릴리즈 규칙

- **`node_modules/` 커밋 금지**: `.gitignore`에 등록된 항목은 절대 커밋하지 않습니다.
- **히스토리 무결성**: 공유 브랜치에 force push하지 않습니다.
- **GitHub Actions**: `main` 브랜치 푸시 시 자동 배포됩니다.

---

## 6. 🤖 AI 에이전트 지침

> [!IMPORTANT]
> **이 레포에는 `agents/` 디렉토리가 없습니다.**
> 공개 사이트 특성상 로컬 에이전트 설정이 별도로 존재하지 않습니다.
> 작업 시 **전역 규칙**(`~/.gemini/config/AGENTS.md`) 및 **GoVail 조직 공통 원칙**을 따르세요.

**적용 공통 원칙:**
- 사내 인프라 정보(IP, 호스트명, VPN)를 공개 파일에 포함하지 않는다.
- 콘텐츠 변경 전 `govail-docs` 레포의 최신 아키텍처 문서를 참조한다.
- 마케팅 과장 표현을 사용하지 않는다.
